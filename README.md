🛒 E-commerce QA Manual Testing Project

📌 Project Overview

This project focuses on performing manual testing for an E-commerce website. The goal is to ensure that the application functions correctly, is user-friendly, and free from critical defects.

---

🎯 Objectives

- Validate core functionalities of the E-commerce website
- Identify and report bugs
- Ensure smooth user experience
- Verify UI and functional requirements

---

🧪 Testing Scope

The following modules were tested:

- User Registration
- Login / Logout
- Product Search
- Add to Cart
- Checkout Process
- Payment Flow

---

🛠️ Tools & Technologies

- Manual Testing
- JIRA (Bug Tracking)
- MS Excel (Test Cases)

---

📂 Project Files

- 📄 Test Cases → Excel file
- 📄 Bug Report → Excel / PDF
- 📄 Project Report → PDF

---

🧾 Test Scenarios Covered

- Valid & Invalid Login
- Product search using keywords
- Adding/removing items from cart
- Order placement
- Payment validation

---

🐞 Bug Reporting

Bugs were documented with:

- Bug ID
- Description
- Steps to Reproduce
- Expected vs Actual Result
- Severity & Priority

---

📊 Results

- Total Test Cases: 33
- Passed: 14
- Failed: 19
- Bugs Found: 19

---

🚀 Conclusion

The testing process helped identify key issues and improve the quality of the application. Automation testing can be implemented in the future for regression testing.
